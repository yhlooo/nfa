## Context

nfa 项目已有 PolyMarket 相关功能：
- `pkg/polymarket/`: PolyMarket API 客户端、市场监听器 (Watcher)
- `pkg/ui/polymarket/`: 市场浏览 TUI

需要新增策略交易系统，在现有监听能力基础上实现自动化交易。

关键约束：
- 策略必须无状态、无副作用，可在任何时候调用
- 策略输出订单数量最少价值 1 美元，执行器可按比例放大
- 支持模拟交易 (dry-run) 和真实交易（预留）
- 价格历史数据最多保留 1000 个点

## Goals / Non-Goals

**Goals:**
- 定义清晰的策略接口，策略接收市场和持仓信息，输出交易决策
- 实现策略执行器，在指定市场上运行策略
- 支持 dry-run 模式，模拟订单创建和成交
- 预留真实交易接口（空实现，TODO）
- 实现交易 UI，实时展示持仓、现金、交易记录
- 新增 CLI 命令入口

**Non-Goals:**
- 复杂策略实现（仅需简单示例策略）
- 策略持久化存储
- 多市场同时交易
- 策略回测

## Decisions

### 1. 策略接口设计

**Decision**: 策略接口包含 `Name()` 和 `Execute()` 方法

```go
type Strategy interface {
    Name() string
    Execute(ctx context.Context, in Input) (*Result, error)
}
```

**Rationale**: 简单清晰的接口，策略无状态、无副作用，便于测试和扩展。

### 2. 执行器与 Watcher 的关系

**Decision**: 执行器内部创建并管理 Watcher，不复用外部 Watcher

**Rationale**: 执行器需要完整的生命周期管理，包括启动、停止、事件处理。独立管理 Watcher 更清晰。

### 3. 价格历史管理

**Decision**: 使用环形缓冲区，最多保留 1000 个价格点

**Rationale**: 限制内存使用，1000 个点足够策略分析近期趋势。

### 4. 订单归属与跟踪

**Decision**: 执行器内部跟踪所有订单，不依赖外部订单查询

**Rationale**: 执行器启动时指定市场，所有订单都在该市场上进行，执行器内部记录即可。

### 5. dry-run 模拟成交

**Decision**: 当市场价格触及限价单价格时模拟成交

**Rationale**: 简单有效，模拟真实市场行为。

### 6. CLI 命令格式

**Decision**: `nfa tools polymarket trade --dry-run -m <slug> -s <strategy> -x <multiplier>`

**Rationale**: 符合现有命令风格，参数清晰。

### 7. 资产总价值计算

**Decision**: `总价值 = 现金 + Yes持仓 * Yes买价 + No持仓 * No买价`

**Rationale**: 以当前市场价格计算持仓价值，反映实时盈亏。

## Risks / Trade-offs

**Risk**: dry-run 模拟成交与真实市场可能有差异
→ **Mitigation**: 文档说明 dry-run 仅供策略验证，真实交易需谨慎。

**Risk**: 真实交易 API 未实现
→ **Mitigation**: 预留接口，TODO 标记，后续补充。

**Trade-off**: 策略无状态
→ 简化实现，但策略无法记忆历史状态。所有必要信息通过 Input 传入。

**Trade-off**: 单市场交易
→ 简化实现，后续可扩展多市场。
